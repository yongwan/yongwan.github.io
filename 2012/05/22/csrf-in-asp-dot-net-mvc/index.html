Cross-Site Request Forgery(CSRF) in ASP.NET MVC

22 May 2012

한국어로는 사이트 간 요청 위조. 정상적으로 생성된 쿠키를 이용해서 특정 사이트를 공격하는 방법. 이 때문에 Request를 하는 User를 확인하는 과정이 필요한데 ASP.NET MVC에서 이를 확인하는 API가 존재.

그거슨 AntiForgeryTokenHelper. 아래 사이트에 자세하게 잘 나와있다.

Prevent Cross-Site Request Forgery (CSRF) using ASP.NET MVC’s AntiForgeryToken() helper

간단하게 말하면 aspx의 form에는 Html.AntiForgeryToken()을 사용하고

<% using(Html.Form("Item", "Submit")) { %>
    <%= Html.AntiForgeryToken() %>
    <!-- etc -->
<% } %>

Controller의 해당 POST action에는 ValidateAntiForgeryTokenAttribute를 사용하면 된다.

[HttpPost]
[ValidateAntiForgeryToken]
public ViewResult Submit()
{
    // etc
}

이렇게 공격하겠다고 생각한 해커도 대단한 듯;;
